Bug Descriptions Bug #01: Server Error on Concurrent Deletion (Race Condition) ID:  BUG-001 (Related to TC_CART_04) Severity:  Medium/High Steps to Reproduce:  Open the same Cart in two different browser tabs -&gt; Click &quot;Delete&quot; on Tab A -&gt; Immediately click &quot;Delete&quot; on Tab B. Expected Result:  Tab A deletes the item; Tab B refreshes or shows &quot;Item already removed.&quot; Actual Result:  Tab B throws a &quot;500 Internal Server Error&quot; because the application attempts to delete an ID that no longer exists in the database. 3. Observations and Recommendations Observations: Standard Flow:  The core functionality (Adding/Removing items) works perfectly for guest users, showing good session management for non-logged-in states. Concurrency Issues:  The system is vulnerable to rapid inputs (spamming) and concurrent actions (multi-tab usage), suggesting a lack of request debouncing and proper exception handling on the backend. Performance:  The NVMe SSD product page loads quickly, but the cart calculation lags slightly when handling multiple updates simultaneously. Recommendations: Implement Debouncing/Throttling:  Add a delay (e.g., 300ms) or a loading state to the &quot;+&quot; quantity button to prevent users from sending too many requests before the previous one finishes. Graceful Error Handling:  Modify the delete API to return a  404 Not Found  or a success message even if the item is already gone, rather than allowing a  500 Server Error . UI Feedback:  Add a small &quot;Updating...&quot; toast or spinner when the cart total is recalculating to improve user perception of speed.
